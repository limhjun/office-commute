# WORKS

README의 `## 주요 기능`을 기준으로, 각 기능을 하나씩 점검하고 개선/업그레이드한다.

## 진행 방식
각 항목마다 다음 단계를 거친다.
1. **현황 파악** — 관련 controller / service / domain / repository / test 코드를 읽고 동작을 정리한다.
2. **진단 리포트** — 제대로 동작하는지, 누락/허점/개선 여지가 무엇인지 사용자에게 보고한다.
3. **방향 합의** — 사용자가 개선 범위와 우선순위를 결정한다.
4. **구현 & 검증** — 좁은 범위로 수정하고, 가장 가까운 Gradle 테스트 태스크로 확인한다.
5. **마무리** — 변경사항 요약, 다음 항목으로 이동.

## 작업 항목

### 1. 직원 및 팀 관리
- [ ] **① 팀 등록 및 조회**
  - 팀 이름, 관리자, 소속 인원을 등록하고 전체 조회한다.
- [ ] **② 직원 등록 및 조회**
  - 직원 기본 정보(이름, 역할, 입사일, 생일)와 소속 팀을 등록하고 전체 목록을 조회한다.

### 2. 출퇴근 및 근무 시간 관리
- [ ] **③ 출퇴근 시간 기록**
  - 직원 ID 기준으로 출근/퇴근 시간을 기록한다.
- [ ] **④ 월별 근무 시간 조회**
  - 특정 직원의 월별 근무를 날짜별로 조회하고, 해당 월 총 근무 시간을 분 단위로 합산한다.

### 3. 연차 관리
- [ ] **⑤ 연차 신청**
  - 하루 단위 연차 신청, 즉시 반영.
- [ ] **⑥ 팀별 연차 정책 적용**
  - 팀마다 'N일 전 등록' 규정을 설정하고, 신청 시점에 검증한다.
- [ ] **⑦ 남은 연차 조회**
  - 직원이 자신의 남은 연차 수를 확인한다.

### 4. 초과 근무 정산 및 리포트
- [ ] **⑧ 초과 근무 정산 + 공휴일 연동 + Excel 리포트**
  - 공휴일/주말을 제외한 월별 법정 근무 시간 기준으로 초과 근무를 분 단위 자동 계산한다.
  - 공공데이터포털 특일 정보 API를 연동하여 공휴일을 반영한다.
  - 월별 초과 근무(직원명, 부서명, 초과 근무 시간, 수당) Excel(.xlsx) 리포트를 다운로드한다.

## 규칙
- 한 번에 한 항목씩 진행한다.
- 항목별로 진단 리포트 → 사용자 승인 → 구현 순서를 지킨다.
- 변경은 작은 단위로 유지하고, 기존 패턴/네이밍을 우선 따른다.
- 스키마 변경은 Flyway 마이그레이션으로 추가한다.
- 항목 완료 시 위 체크박스를 `[x]`로 갱신한다.
